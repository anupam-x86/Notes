Overall, this is a **good solution**. It's modular, readable, and you're practicing functions, which is exactly what you should be doing at this stage.

I'd give it around **8.5/10**. Here are a few improvements.

---

## 1. Don't use the ternary to return a `bool` (most important) [Fixed]

You have:

```cpp
return (number % 2 == 0) ? true : false;
```

But `number % 2 == 0` **already evaluates to a bool**.

Just write:

```cpp
return number % 2 == 0;
```

Same for:

```cpp
return number % 3 == 0;
return number % 5 == 0;
```

---

## 2. `ispositive()` should probably return a `bool` [Fixed]

Instead of:

```cpp
std::string ispositive(int number)
```

I'd write:

```cpp
bool isPositive(int number)
{
    return number > 0;
}
```

Then in `main()`:

```cpp
std::string numPositive { isPositive(number) ? "Yes" : "No" };
```

The reason is that functions should usually answer **one question**.

> Is it positive?

Answer:

```cpp
true
false
```

Formatting ("Yes"/"No") is the job of `main()`.

---

## 3. Zero [Fixed]

Your assignment says:

> Print whether it is positive.

Zero is **not positive**.

So I'd simply print:

```
No
```

instead of

```
Zero
```

unless the assignment explicitly wants to distinguish zero.

---

## 4. Naming

I'd use camelCase:

```cpp
isEven()
isPositive()
isDivBy3()
isDivBy5()
```

This is just style.

---

## 5. Repeated function calls

You do:

```cpp
isdivby3(number)
```

twice.

Instead:

```cpp
bool div3 { isDivBy3(number) };
bool div5 { isDivBy5(number) };

std::string numDivBy3 { div3 ? "Yes" : "No" };
std::string numDivBy5 { div5 ? "Yes" : "No" };
std::string numDivBy3And5 { (div3 && div5) ? "Yes" : "No" };
```

Now each function is called only once.

---

## 6. Nice observation

I liked this:

```cpp
std::string numDivBy2 {numEven};
```

Since every even number is divisible by 2, you're reusing the result instead of checking again.

---

## 7. Another possible design

You could avoid storing `"Yes"`/`"No"` strings entirely:

```cpp
std::cout << "Is Even: " << (isEven(number) ? "Yes" : "No") << '\n';
```

This becomes useful later because you don't create unnecessary variables.
