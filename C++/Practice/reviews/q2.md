## 1. Remove `const` from pass-by-value parameters (optional)

You currently have:

```cpp
int threeIntSum(const int firstNum, ...)
```

This isn't wrong.

But since these are copied into the function, many C++ style guides simply write:

```cpp
int threeIntSum(int firstNum, int secondNum, int thirdNum)
```

because the caller can't observe whether the local copies are `const`.

If you like using `const` to remind yourself not to modify parameters, that's fine at this stage.

---

## 2. Function name

```cpp
threeIntComparison()
```

The name says **how** you're doing something (comparison), not **what** the function accomplishes.

Something like:

```cpp
printLargestAndSmallest()
```

would tell me the purpose immediately.

Not required, just something to think about.
