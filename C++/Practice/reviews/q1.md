# ⚠ Things I'd improve

---

## 1. Variable naming

```cpp
int Age {};
```

This is the only variable that starts with a capital letter.

Everything else uses

```cpp
firstName
lastName
nextYearAge
```

This one should be

```cpp
int age {};
```

Consistency matters.

---

## 2. Function name

```cpp
printStrSentence()
```

This feels awkward.

It isn't printing "a string sentence."

It's greeting the user.

Better names:

```cpp
printGreeting()
```

or

```cpp
displayGreeting()
```

or

```cpp
greetUser()
```

Good names explain intent.

---

## 3. Parameter names

```cpp
firstNameView
```

Since the type is already

```cpp
std::string_view
```

the name doesn't need to repeat that.

Better

```cpp
firstName
```

```cpp
lastName
```

The type tells us it's a view.

---

## 4. This comment

```cpp
Declared printStrSentence which gonna define after main function for better visiblilty
```

I'd simply write

```cpp
// Forward declaration
```

or

```cpp

```

Shorter.

Professional.

## 5. Prompt formatting

Instead of


* consistent capitalization
* easier to read
* natural English

Same for

```
last name

```

# 🤔 Design discussion

You computed

```cpp
const int nextYearAge { age + 1 };
```

Good.

But ask yourself:

Do we actually need a variable?

This

```cpp
std::cout << "Next year you will be "
          << age + 1
          << '\n';
```

is also perfectly fine.

I actually like your version more because

* it gives the calculation a name
* easier to debug later
* readable

So I'd keep it.
